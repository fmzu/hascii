# hascii

ドット絵を描いて共有できるWebアプリ。核となる価値は「描いた絵の状態がそのままURL文字列になる」こと。バックエンド不要で「描く→URLを送るだけで見せられる」。

## 解決する課題

1. **誰の・どんな課題を解決するのか**
   - 新規訪問者がログインしないと描き始められない。正確には既存の `/dot` エディタ自体はログイン不要だが、トップページ・ヘッダに導線がなく発見できない
   - 共有を受け取った側もログインしないと投稿詳細を見られない（`GET /posts/:post` が `verifyAuth()` で401を返す）
   - 描いた絵をアプリ外（ブログ・SNS等）で使う手段がURLコピーしかない
2. **なぜ既存の状態では不十分か**
   - 「URLに絵の状態を持たせてサーバーレスで見せられる」のが本アプリの強みなのに、閲覧に認証を要求してしまい強みを自ら殺している
   - さらに `GET /posts`（一覧API）が `isPublic` でフィルタしておらず、非公開に設定した投稿が全件レスポンスに含まれてしまうプライバシーバグがある。放置するとユーザーが「非公開」にした絵が第三者から見える状態が続く
3. **導入前と導入後の体験**
   - 導入前: トップページに来た初見ユーザーは何をすればいいか分からずログイン画面に誘導される → 離脱。共有URLを受け取った人はログインを求められて見られずに離脱
   - 導入後: トップページから「描いてみる」で即座にキャンバスが開く。投稿はログイン時のみ促され、未ログインでも共有URLの絵はそのまま閲覧できる。非公開投稿は一覧に出ない
4. **放置したときの損失**
   - 創作系サービスは「試すまでの摩擦」がそのまま離脱率に直結する。導線不備は新規流入をほぼゼロにする
   - 非公開フィルタ漏れは信頼を損なうプライバシーインシデントであり、優先度が最も高い

## 機能一覧

- トップページ・ヘッダからログイン不要で `/dot` エディタに入れる導線
- `/dot` エディタからの投稿（ログイン時はダイアログでタイトル・公開設定を入力、未ログイン時はログイン誘導ダイアログ）
- サインイン/サインアップページの `callbackUrl` パラメータ対応（投稿前にいた `/dot/xxxx` に戻す）
- 投稿詳細の匿名閲覧（非公開投稿は所有者以外に404）
- 投稿一覧APIの非公開フィルタ修正
- `/dot` エディタからのエクスポート（PNG保存 / ドット文字列コピー）

## 画面 / CLI仕様

### ①導線整備

- トップページに「描いてみる」ボタンを追加し、押下で `/dot`（空キャンバス）へ遷移する
- ヘッダー（`ui/app/routes/_main/components/main-header.tsx`）: 未ログイン時にも「作成」ボタンを表示し `/dot` へ遷移させる。ログイン済み時の既存導線（`/my/posts` への「作成」ボタン・`AvatarPopover`）は変更しない

### ②投稿時ログイン誘導

- `/dot` エディタ（`DotWorkspace`、`ui/app/routes/_main.dot._index/components/dot-workspace.tsx`）に「投稿」ボタンを追加する
- ヘッダー内の並びは `[エクスポート ▾] [投稿]` とする（投稿はサーバーへ公開する主操作のため右端に配置し目立たせる）
- ログイン済みの場合: タイトル・公開設定を入力するダイアログを開き、確定で `POST /posts` を呼んで投稿する
- 未ログインの場合: 「投稿にはログインが必要です」という内容のダイアログを表示し、サインイン/サインアップへの導線を示す
  - 現在の `/dot/xxxx` のURLを `callbackUrl` としてサインイン/サインアップページに渡し、ログイン後に同じ絵の画面へ戻す
  - 絵の状態はURLそのものが持つため、ログイン前後で絵データを引き継ぐ処理は不要（URLに戻すだけでよい）
- サインイン（`/sign/in`）・サインアップ（`/sign/up`）ページに `callbackUrl` クエリパラメータを受け取り、認証成功後にそのURLへリダイレクトする処理を追加する

### ③閲覧開放（API）

- `GET /posts/:post`（`api/interface/routes/post.ts`）から `verifyAuth()` を外し、匿名でも呼び出せるようにする
- 未ログイン時の分岐（`authUserEmail === null` で `isMine: false` を返す処理）は既存コードにすでに実装済みだが `verifyAuth()` によって未ログインリクエストが401で弾かれるため事実上のデッドコードになっている。`verifyAuth()` を外すことでこの分岐が生きる
- **非公開投稿（`isPublic: false`）は所有者以外からのリクエストに対して404を返す**（存在有無を漏らさないため403ではなく404とする）
- `PUT` / `DELETE` / いいね関連 / `my/*` 系のエンドポイントは引き続き認証必須のまま変更しない

### ④非公開漏れ修正（API）

- `GET /posts`（`api/interface/routes/posts.ts` の一覧取得）に `isPublic = true` かつ `isDeleted = false` の条件を追加し、非公開・削除済みの投稿を返さないようにする

### ⑤エクスポート機能

- `/dot` エディタに「エクスポート」ドロップダウンメニューを追加する
- **PNG保存**: canvasからスケールを指定（1ドット=16px程度）して画像を生成し、ブラウザのダウンロードとして保存する。サーバー通信なしでクライアント完結する
- **ドット文字列コピー**: URL共有の元になっているドット文字列をクリップボードにコピーする
- 各出力形式は「グリッドデータ → 出力データ」に変換する独立した純関数として実装し、1関数1ファイルで分離する。将来的に別形式（例: CSS出力）を追加しやすい構造にする

## データ構造

既存の `schema.posts`（`api/drizzle.schema.ts`）をそのまま利用する。今回のスコープで新規テーブル・カラム追加はない。

```typescript
// 参考: 既存スキーマの主要フィールド（変更なし）
type Post = {
  id: string
  userId: string
  name: string
  dots: string        // ドット絵の状態を表す文字列（URLの一部になる）
  title: string
  description: string
  regulation: string
  isPublic: boolean
  isDeleted: boolean
}
```

## API

| メソッド | パス | 変更内容 |
|---------|------|------|
| GET | /posts | `isPublic = true` かつ `isDeleted = false` のフィルタを追加（④） |
| GET | /posts/:post | `verifyAuth()` を撤去し匿名閲覧可に。非公開投稿は所有者以外へ404（③） |
| POST | /posts | 変更なし（認証必須のまま） |
| PUT | /posts/:post | 変更なし（認証必須のまま） |
| DELETE | /posts/:post | 変更なし（認証必須のまま） |

## 技術構成

- 言語: TypeScript
- ランタイム: Cloudflare Workers
- 構成: モノレポ（`ui/` = React Router v7 + Cloudflare Workers、`api/` = Hono + Drizzle + Cloudflare D1）
- 認証: Auth.js Credentials（メール + パスワード）
- その他: パッケージマネージャは Bun、Lint/Format は Biome

## テスト観点

- 正常系:
  - 未ログインで `/dot` にアクセスして描画・URLが更新される
  - ログイン済みで投稿ダイアログからタイトル・公開設定を入力し `POST /posts` が成功する
  - 未ログインで投稿を試みるとログイン誘導ダイアログが表示され、ログイン後に元の `/dot/xxxx` へ戻る
  - 未ログインで公開投稿の詳細（`GET /posts/:post`）が閲覧できる
  - PNG保存・ドット文字列コピーがそれぞれ期待通り出力される
- 異常系:
  - 未ログインで非公開投稿の詳細を取得すると404が返る
  - 所有者以外のログインユーザーが他人の非公開投稿を取得しても404が返る
- エッジケース:
  - `GET /posts` の一覧に非公開投稿・削除済み投稿が含まれない
  - `callbackUrl` が不正な値・外部ドメインの場合にオープンリダイレクトにならない
  - PNG保存はグリッドの純関数部分にユニットテストを書き、キャンバス描画自体はブラウザ依存のためロジックとレンダリングを分離してテストする

## 実装順序

リスクが高く独立性の高いものから着手する。

1. ④ 非公開漏れ修正（プライバシーバグのため最優先）
2. ③ 閲覧開放（API）
3. ① 導線整備
4. ② 投稿時ログイン誘導
5. ⑤ エクスポート機能

## バックログ（今回やらない）

- **CSS出力形式**: box-shadow等でドット絵をCSS化し、他プロジェクトへ流用できるようにする。エクスポートメニューに形式を1つ追加するだけで対応できる設計にしてある
- **OGP画像生成**: SNS共有時にドット絵がカードに表示されるようにする
- **リミックス機能**: 他人の公開作品をエディタで開いて自分の版を作れるようにする
- **OAuth移行**: GitHub/Google認証への対応。自前パスワード管理からの脱却

## 決定ログ

- 一覧カードのタイトル部分の下側だけ角丸なのは、小学生の絵の宿題に貼る「タイトルと名前を書く紙」のオマージュ。バグではなく意図的なデザイン
- 未ログインでのサーバーへの投稿（匿名投稿）は意図的に非対応とする。匿名投稿を許すとスパム対策コストが発生し、`posts.userId` が `notNull` のDBスキーマ変更も必要になるため。代替手段としてローカルへのエクスポート（PNG保存等）を用意する
- サーバーへ置く操作は「投稿」、ローカルへ出す操作は「エクスポート」と呼び分ける。「保存」という語はボタン名に使わない（PNG保存との混同を避けるため）
